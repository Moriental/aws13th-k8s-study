### 내가 설정한 것
Gateway 생성
```
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: web-gateway
  namespace: default
spec:
  gatewayClassName: nginx
  listeners:
  - name: https
    protocol: HTTPS
    port: 443
    hostname: gateway.web.k8s.local
    tls:
      mode: Terminate
      certificateRefs:
      - kind: Secret
        name: web-cert
    allowedRoutes:
      namespaces:
        from: Same
```

HTTPRoute 생성
```
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: web-route
  namespace: default
spec:
  parentRefs:
  - group: gateway.networking.k8s.io
    kind: Gateway
    name: web-gateway
  hostnames:
  - gateway.web.k8s.local
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /
    backendRefs:
    - name: web
      port: 80
```

### 트러블 슈팅
1. 공식 양식에는 없지만 문제를 풀기 위해 필요한 요소를 추가해야 함.
```
tls:
      mode: Terminate
      certificateRefs:
      - kind: Secret
        name: web-tls-secret
```
2. 기존 인그레스는 삭제했지만 로컬 주소록(/etc/hosts)에는 여전히 인그레스의 주소로 등록되어 있어 404 에러 뜸. 이것만 게이트웨이 주소로 바꿔주면 해결.

### 새롭게 안 것
인그레스: 파일 하나에 도메인, 경로, TLS 인증서 설정을 짬뽕으로 다 적어서 관리
GatewayAPI: 클래스, 인증서(TLS), 리스너를 관리하는 Gateway와 리스너를 통해 들어온 트래픽 중 특정 도메인을 목적지 서비스로 보내는 HTTPRoute로 구분
GatewayClass: 이 게이트웨이를 쓸 때 어떤 엔진을 쓸지 명시해둔 것
