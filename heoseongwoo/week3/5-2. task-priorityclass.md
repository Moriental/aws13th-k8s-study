## 풀이 과정

### 1. priorityclass의 우선순위 체크(시스템 밖에 없긴 함)
```
[cka0001@k8s-master ~]$ kubectl get priorityclass
NAME                      VALUE        GLOBAL-DEFAULT   AGE     PREEMPTIONPOLICY
system-cluster-critical   2000000000   false            7d11h   PreemptLowerPriority
system-node-critical      2000001000   false            7d11h   PreemptLowerPriority
```

### 2. high-priority 생성
```
[cka0001@k8s-master ~]$ vi high-priority.yaml

apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority
value: 9
globalDefault: false
description: "second priority ang gimotti"
```

### 3. busybox-logger에 우선순위 부여
```
[cka0001@k8s-master ~]$ kubectl edit deployment busybox-logger -n priority

priorityClassName: high-priority
```
